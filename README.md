# Project

> 一个结构清晰、协作友好的软件产品项目。

---

## 项目定位

本项目遵循 **文档驱动、代码隔离、素材分类** 的组织原则，致力于打造一个维护性强、上手成本低的产品代码仓。无论是独立开发还是团队协作，目录结构都能保证信息不散落、资源不混乱。

---

## 目录一览

```
/
├── docs/                   # 产品文档：需求、架构、决策纪要
├── src/                    # 业务代码：所有源码的唯一入口
├── assets/
│   ├── design/             # 设计素材：UI 稿、原型、品牌资源
│   ├── bug/                # 问题截图：Bug 复现、报错截图
│   └── reference/          # 参考灵感：竞品、参考设计
├── notes/                  # 开发笔记：踩坑记录、技术总结
├── AGENTS.md               # AI 开发助手上下文说明
└── README.md               # 项目说明（本文件）
```

---

## 协作与开发入门

### 快速开始

```bash
# 克隆仓库
git clone <repo-url>
cd <project-dir>

# 进入源码目录
cd src

# 根据项目技术栈安装依赖（示例）
npm install      # Node.js
# pip install -r requirements.txt   # Python
```

### 目录约定

| 你要做的事 | 文件放哪里 |
|-----------|-----------|
| 写业务代码 | `/src` |
| 写产品文档 / 技术方案 | `/docs` |
| 存设计稿 / 原型图 | `/assets/design` |
| 存 Bug 截图 | `/assets/bug` |
| 存参考图 / 灵感 | `/assets/reference` |
| 写踩坑笔记 / 技术总结 | `/notes` |

### AI 协作

本项目已配置 `AGENTS.md`，接入 AI 编码助手时会自动加载上下文和规则。核心原则：

- **代码只在 `/src` 写**
- **文档一律归 `/docs`**
- **素材分类进 `/assets`**
- **笔记集中放 `/notes`**
